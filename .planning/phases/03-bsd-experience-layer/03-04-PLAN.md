       - 'use client' component
       - Props: { triggerSelector: string; className?: string } -- triggerSelector is the CSS selector for the section boundary element
       - Implementation approach: CSS-only ink wash effect using clip-path animation driven by GSAP ScrollTrigger scrub (no sprite sheet needed -- generate the organic ink feel with multiple layered clip-paths and CSS filters)
       - Structure:
         - A fixed/absolute overlay div with pointer-events: none (CRITICAL: never block clicks)
         - Multiple ink "tendril" layers (3-4 div elements) with different clip-path animations
         - Each tendril has a dark ink color (#0a0e1a with 0.7-0.9 opacity) and blur filter
       - GSAP ScrollTrigger scrub config:
         - trigger: the section boundary element
         - start: "top bottom" (ink starts when next section enters viewport)
         - end: "top center" (ink fully resolves by the time section is centered)
         - scrub: 0.5 (smooth follow)
       - Animation sequence (per tendril):
         - clipPath morphs from edges inward: starts as thin tendrils at edges, expands to cover ~30-40% of viewport, then recedes
         - Opacity peaks at 0.6-0.8 mid-transition, fades to 0 at end
         - Different tendrils animate at slightly different rates for organic feel
       - Scrolling backward reverses the effect naturally (scrub handles this)
       - If reducedMotion: render nothing (return null)
       - If current transition was triggered by nav click (detect via store.currentSection change without scroll): use a quick CSS opacity fade (200ms) instead of ink wash
